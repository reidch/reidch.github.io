# GPU SPH 流体仿真与屏幕空间渲染

这是一个基于 OpenGL 4.5 与 Compute Shader 的实时流体仿真和渲染项目。项目使用光滑粒子流体动力学（SPH）描述流体运动，并将邻域搜索、密度、压力、作用力和粒子更新等主要计算放在 GPU 上执行。

在当前版本中，我保留了 GPU SPH 仿真管线，并重点升级了最终显示方式：不再只把粒子显示为点或独立球体，而是通过多阶段屏幕空间重建，将大量离散粒子合成为具有连续表面、反射、折射和体积吸收效果的流体。

## 当前规模

当前上传版本的配置为：

- 空间网格：`64 × 64 × 64`
- 每个初始网格单元：4 个粒子
- 粒子总数：`64³ × 4 = 1,048,576`
- Compute Shader 工作组大小：256
- 仿真时间步长：`0.003`

这里的粒子总数来自当前代码中的实际常量，而不是估算值。

## GPU SPH 仿真管线

每一帧的仿真由一系列 GPU 计算阶段组成：

1. 为每个粒子计算所在网格单元
2. 统计各网格单元中的粒子数量
3. 计算网格粒子区间的前缀和与偏移
4. 按网格单元重新排列粒子索引
5. 在局部邻域中计算密度
6. 根据状态方程计算压力
7. 计算压力力、黏性力、重力和交互碰撞力
8. 更新粒子位置与速度

基于规则网格的空间划分将邻域查询限制在附近单元，而不是让每个粒子与所有其他粒子进行比较，从而使百万级粒子的实时计算成为可能。

## 密度、压力与作用力

密度阶段使用平滑核函数累加邻域粒子的贡献。

压力使用类似 Tait 状态方程的形式，根据粒子密度与静止密度之间的比例计算：

```text
P = stiffness × ((density / restDensity)^gamma − 1)
```

项目还计算：

- 对称压力力
- 黏性力
- 重力
- 流体边界约束
- 可移动控制球体与流体之间的交互作用

## 碰撞与交互

流体被限制在三维盒状区域中。粒子更新阶段会在恒定加速度假设下求解粒子与盒体平面的碰撞时间，并在一个时间步内处理多次碰撞，以减少高速粒子穿过边界的问题。

场景中还包含一个可移动的控制球体。它会对附近粒子施加接触响应，因此可以用于推动、分离和扰动流体。

## 屏幕空间流体渲染

屏幕空间流体渲染由多个阶段构成。

### 1. 场景颜色与深度

首先将非流体场景渲染到 HDR 颜色纹理和 32 位深度纹理中，为后续遮挡、折射与合成提供输入。

### 2. 粒子深度重建

每个 SPH 粒子以面向相机的点精灵绘制。Fragment Shader 根据圆盘坐标解析计算球面前表面，并写入：

- 真实球面深度
- 线性观察空间深度
- `gl_FragDepth`

这样能够正确处理粒子与场景以及不同粒子之间的深度关系。

### 3. 厚度累积

另一个渲染阶段将每个粒子穿过视线方向的弦长累积到浮点厚度纹理中。厚度随后用于控制流体的吸收、颜色与折射强度。

### 4. 双边深度平滑

原始粒子深度会留下明显的粒子凹痕。项目使用横向和纵向分离的双边滤波，在平滑表面的同时尽量保留轮廓与深度不连续边界。

### 5. 法线重建与最终合成

最终 Fragment Shader 从平滑后的深度重建观察空间位置与表面法线，并结合：

- 环境立方体贴图反射
- 屏幕空间场景折射
- Schlick Fresnel
- Beer–Lambert 吸收
- 流体内部散射颜色
- 窄范围镜面高光
- 薄边缘亮度增强

将离散粒子合成为具有连续水面外观的实时流体。

## 渲染模式对比

程序支持三种显示模式，可在运行时切换：

- 屏幕空间连续流体
- 实例化球体
- 粒子点

按 `C` 键可以循环切换三种模式。这不仅便于调试，也能直接观察从原始粒子数据到连续流体表面的视觉变化。

## 操作方式

- `C`：切换屏幕空间流体、实例化球体和点模式
- `V`：显示或隐藏天空盒
- `W/A/S/D` 与鼠标：移动相机
- `J/L`、`H/Y`、`I/K`：沿三个轴移动控制球体

## 项目意义

这个项目将 GPU 并行仿真与现代实时流体渲染连接在同一管线中。它不仅关注“如何让大量粒子运动”，也关注如何从粒子数据恢复可读、连续且具有材质感的流体表面。
